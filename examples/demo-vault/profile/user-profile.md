## 5. How Gallup shapes collaboration

### Decision style

Strategic #4 + Analytical #11 + Context #8 = **"show me data, let me see the pattern, I decide."** But Deliberative #32 is low — decides fast. Wants data for speed, not for insurance. Best format: **1 recommendation with evidence**, not a menu of 5 options.

### What energizes

- Measurable progress (Achiever #5 + Competition #3) — show benchmarks, comparisons, scores. "Better than X" resonates.
- New ideas and connections (Ideation #2) — novel insights spark engagement.
- Fixing what's broken (Restorative #1) — present problems with diagnosis, not just symptoms.
- Understanding WHY (Context #8) — history and reasoning matter when relevant.

### What demotivates

- **No visible progress** — did 10 things, feels like 0. Long research without artifacts drains energy.
- **Artificial consensus** — smoothing edges instead of direct answers (Harmony #34 = lowest).
- **Routine without novelty** — mechanical tasks should be automated, not discussed.
- **Bureaucracy without function** — process must serve outcome, not exist for itself.

### How to deliver bad news

Restorative #1 + Command #14 = **wants to know what's broken, fast.** "Plan doesn't work, here's why, here's what to do" beats "there are some small challenges we might consider..."

### Showing progress

Achiever #5 needs tangible output daily. In research phases — show intermediate artifacts. "Today: 3 tasks closed, pipeline v5 ready" is not flattery, it's an objective anchor against Achiever's chronic "not enough."

## 6. Shadow sides & failure modes

### Strengths overplayed (Top 10)

**Restorative #1 — "Everything's broken, let me fix it":** sees problems where there aren't any. Working solution feels "unfixed". Pulls toward disassembling and reassembling what already works.

**Ideation #2 — "Oh, and here's another idea!":** scope creep in pure form. New idea at every step. Can abandon current for something more interesting.

**Competition #3 — "They have it better":** can fixate on competitor instead of own product. Comparison as stress source, not just motivation.

**Strategic #4 — "I see 5 paths... and choose none":** paradox of choice. "Too early to decide, need more data" — when data is already sufficient.

**Achiever #5 — "Didn't do enough today":** chronic self-dissatisfaction. Burnout risk.

**Includer #6 — "Let's add everyone":** in AI context — pull to add more models to consilium, more agents, more sources. More voices ≠ better decision.

**Developer #7 — "It'll grow, give it a chance":** can give too many chances to a tool / model that isn't performing.

**Context #8 — "First, understand the history":** can dive into deep past-research instead of acting in present.

**Individualization #9 — "Every case is unique":** resists templates and automation.

**Arranger #10 — "Let me reorganize everything better":** refactoring for its own sake.

### Bottom-5 gaps (30-34)

**Relator #30 + Harmony #34 (both low):** can be blunt without realizing impact. Leader, not facilitator.

**Maximizer #31 (low):** doesn't polish. "Works → next." May underinvest in what already works well.

**Deliberative #32 + Adaptability #33 (both low):** not cautious + not flexible to sudden changes = dangerous combo. Decides fast → plan is rigid → when reality shifts, may dig into the old plan.

### The core tension

**Ideation #2** (generates new) vs **Focus #13** (needs one goal) vs **Achiever #5** (needs progress).

Eternal pull: wants novelty, needs focus, pressured for results. **The workflow pipeline is literally an exoskeleton for managing this conflict.** Framework prevents Ideation from destroying Focus, and Achiever sees checkboxes.

**→ The pipeline is not bureaucracy — it's a discipline prosthetic. Protect it, don't simplify it.**

## 7. Active shadow coaching — when to intervene

The agent SHOULD flag these patterns when observed. Short, direct. Not every time — only when the pattern is costing real time, money, or progress. Format: one sentence naming the pattern + one sentence with the actionable question.

### Trigger table

| I observe...                                                                | Shadow pattern                      | What I say                                                                          |
| --------------------------------------------------------------------------- | ----------------------------------- | ----------------------------------------------------------------------------------- |
| 3rd idea in a session unrelated to current task                             | **Ideation #2 scope creep**         | "That's Ideation — backlog or switch tasks?"                                        |
| Wants to redo / refactor something that works fine                          | **Restorative #1 phantom fix**      | "What specifically is broken? It works now — is there a real bug?"                  |
| Stuck choosing between 2+ approaches, asks for more data when enough exists | **Strategic #4 analysis paralysis** | "Data is sufficient. My recommendation: X, because Y. Going with it?"               |
| Adds 6th+ model to consilium or wants "ask one more"                        | **Includer #6 noise creep**         | "5 models already answered, more voices won't add signal. Synthesizing?"            |
| Keeps tool / model / approach that failed 2+ times                          | **Developer #7 sunk cost**          | "This is the second fail for X. Suggest replacing with Y — data supports the swap." |
| End of session, did a lot but says "we did little"                          | **Achiever #5 blindness**           | "Objective session output: [list of completed items]. That's N tasks."              |
| Wants to reorganize framework / pipeline / structure that works             | **Arranger #10 meta-work**          | "Current structure works. Will reorganization give a concrete gain?"                |
| Deep dive into history / context when action is clear                       | **Context #8 rabbit hole**          | "Context is clear, we can move. Or is there a specific question about history?"     |
| Says "this niche / task / case is special, can't batch"                     | **Individualization #9 anti-batch** | "Batch covers N% of cases. Here are the exceptions separately."                     |
| Competitor teardown turns into "they're so far ahead"                       | **Competition #3 demotivation**     | "They have a gap in X. That's our entry point, not their strength."                 |
| Decides fast on something with irreversible consequences                    | **Deliberative #32 gap**            | "This is irreversible: [consequence]. 30 seconds — sure?"                           |
| Plan broke but keeps pushing the original plan                              | **Adaptability #33 gap**            | "Context changed: [what changed]. Old plan doesn't account for this. Replan?"       |

### Rules for intervention

- **Frequency:** max 1 flag per session. More = nagging, triggers pushback on principle.
- **Tone:** factual, not patronizing. Name the pattern, ask the question, move on. No lectures.
- **Override:** if they say "I know, do it" after flag — respect it. One flag is enough. The point is awareness, not control.
- **Don't flag:** when in flow and productive. Shadow coaching is for decision moments, not execution.
- **Kill switch:** if no flag changed a decision in 5 sessions — stop flagging, the feature isn't working.

## 8. Work style summary

- Demands rigor: no starting without a plan, no code without agreement
- Values AI autonomy — "do it yourself, don't ask unnecessary questions"
- Controls quality tightly — 70+ feedback memories in ~2 weeks
- Thinks in systems: the workflow framework is an orchestration framework, not a todo list
- Economizes resources: caches everything, rotates accounts, counts API credits
- Hates data loss and scope creep
- **Maximum delegation:** wants AI to do everything end-to-end. Ceiling is "enter password in a dialog" or "click a confirm button." Never say "go to Settings and do X" — do it yourself or script it.
- The 70 feedback rules are a **learned system to compensate for low Deliberative #32**, not natural temperament.
